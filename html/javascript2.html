JavaScript Notes: Primitive string vs String Object

There are two types of "string" in JavaScript, which can be confusing. Here's the breakdown.

  • 1. Primitive string (lowercase)
    • This is the normal, everyday string you create with literals.
    • Example: let myName = "Aarnav";
    • It's a primitive data type, just like number and boolean.
    • Checking its type: typeof myName will return "string".
    • This is the lightweight, fast, and recommended version to use.
  • 2. The String Object (Uppercase)
    • This is a full-blown object wrapper, created using the new keyword.
    • Example: let myNameObj = new String("Aarnav");
    • Checking its type: typeof myNameObj will return "object".
    • You should almost never use this. It's slower and leads to confusing behavior (e.g., "Aarnav" === new String("Aarnav") is false).
  • 3. The "Magic": Autoboxing
    • Question: If myName is just a primitive, how can we call methods on it like myName.toUpperCase()?
    • Answer: Autoboxing.
    • When you try to use a method on a primitive string, JavaScript automatically and temporarily "boxes" it (wraps it) in a String object.
    • It calls the method (e.g., .toUpperCase()) on that temporary object.
    • The method returns the result (the new primitive string "AARNAV").
    • The temporary object is immediately discarded.
  • 4. Java Analogy
    • This is almost identical to int vs. Integer in Java.
    • JavaScript string (primitive) is like Java's int (primitive).
    • JavaScript String (object) is like Java's Integer (wrapper class).
    • The "autoboxing" concept is the same in both languages.

console.log and console.dir are both used for debugging in the browser's console, but they display information, especially objects, in different ways.

The main difference is how they handle DOM elements and JavaScript objects.

console.log( ) is for general-purpose logging.

  • For DOM elements: It prints an interactive HTML-like tree. This is great for seeing the element's structure in the DOM.
  • For JavaScript objects: It usually prints the object's string representation, though most modern browsers now show an expandable object summary.

console.dir( ) is for displaying an interactive, property-list view of an object.

  • For DOM elements: It prints the element as a full-fledged JavaScript object, showing all its properties (like id, children, style, onclick, etc.). This is extremely useful for inspecting the properties of a DOM node.
  • For JavaScript objects: It displays an expandable, JSON-like tree of all the object's properties, which is its primary purpose.

Comparison at a Glance

  • console.log()
    • Best For: General output, viewing DOM structure.
    • Output for a DOM Element: <body class="..."> (HTML-like tree)
    • Output for a JS Object: {a: 1} (String or object summary)
  • console.dir()
    • Best For: Inspecting an object's properties.
    • Output for a DOM Element: body (Expandable list of all JS properties)
    • Output for a JS Object: {a: 1} (Expandable list of properties)
