Truthy vs. Falsy in JavaScript

This is a fundamental concept in JavaScript. In JavaScript, every value has an inherent boolean value, meaning it can be evaluated as either "truthy" or "falsy" in a boolean context.

Key Application: This logic is exactly how JavaScript decides what to do in an if statement and which elements to keep when using the .filter() array method.

Falsy Values üëé

A falsy value is a value that is considered false when encountered in a boolean context. There are only a few falsy values in JavaScript. Both if blocks and .filter() will treat these as false.

  • false
  • 0 (the number zero)
  • -0 (negative zero)
  • "" (an empty string)
  • null
  • undefined
  • NaN (Not a Number)

Truthy Values üëç

A truthy value is a value that is considered true when encountered in a boolean context.

Everything else is truthy.

This includes (but is not limited to):

  • Any non-empty string (e.g., "hello", "0", "false")
  • Any non-zero number (e.g., 1, -5, 3.14, -0.5)
  • Arrays (even empty ones: [])
  • Objects (even empty ones: {})
Method Parameters Description
map() (element, index, array) Returns a new array by transforming elements.
filter() (element, index, array) Returns elements that satisfy a condition.
forEach() (element, index, array) Executes a function on each element (no return array).
reduce() (accumulator, element, index, array) Reduces all elements to a single value.
some() (element, index, array) Checks if any element passes a test (returns boolean).
every() (element, index, array) Checks if all elements pass a test (returns boolean).
find() (element, index, array) Returns the first element that passes a test (or undefined).
findIndex() (element, index, array) Returns the index of the first element that passes a test (or -1).
